Phase 4: Data Pipeline & Custom Uploads

Two distinct flows feed the raster services. Default state layers run through an annual pipeline (The Ray manages this). Custom user uploads are on-demand and async. Neither flow pre-clips rasters — clips are applied on the fly at render time via JSON.

A — Annual Default Pipeline Run 1–2x/year by The Ray (not user-triggered). Loads default state-wide layers, creates one WRO service per state.

Key architectural point (Melissa, 4/13 standup): rasters are NEVER pre-clipped per project or scenario. One WRO service is created per state (defaults) or per custom layer set. The scenario's study area polygon is passed as a clip in JSON at render time — so one stored raster serves every project in that state. The clip JSON is read from the application database at rendering time. VM startup is triggered by the .NET API via Azure Compute API immediately after the upload record is created.
